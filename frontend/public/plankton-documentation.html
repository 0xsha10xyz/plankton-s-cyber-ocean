Plankton's Cyber Ocean — Documentation

This document describes the Plankton app: how to run it, what the frontend and backend do, and how to use the API. For source code and future updates, see the project repository (link at the end). Configuration uses environment files; never share or commit these files to protect security.

1. Getting started

Prerequisites

Node.js 18+ and npm (or yarn/pnpm).

Setup

From the project root: npm install

This installs dependencies for the frontend and backend.

Run the app

CommandWhat it does
npm run devStarts the frontend at http://localhost:8080
npm run dev:backendStarts the backend API at http://localhost:3000
npm run dev:allStarts both frontend and backend

Project structure

frontend/ — Vite + React app (UI, wallet, Swap, Docs).

api/ — Vercel serverless routes at the repo root (same-origin /api/* in production when Root Directory is the repository root).

backend/ — Express + TypeScript API (local dev or VPS).

docs/ — Maintainer documentation (configuration, deployment, integrations).

Configuration uses environment files and hosting dashboards — never commit real keys. See docs/CONFIGURATION.md and SECURITY.md in the repository.

Build

Frontend: npm run build (from root). Backend: npm run build:backend. Output: frontend/dist/ and backend/dist/.

2. Frontend

The Plankton frontend is a single-page app (Vite + React + TypeScript) with wallet connection, account management, and an AI agent chat.

Tech stack

Vite, React 18, TypeScript, Tailwind CSS, shadcn/ui, Framer Motion, React Router, Solana wallet adapter (Phantom, Solflare).

Main features

Navigation — Fixed header with logo; links to Dashboard, Swap, Research, Screener, PAP Governance, Subscription, Roadmap, Docs; Connect Wallet / connected state; Account (when connected); mobile menu. Swap opens the dedicated swap page; other links scroll to sections.

Total Users — The app shows a Total Users count (unique wallets that have connected), in the hero card, stats strip, footer, and on the Swap page. The count updates when you connect and refreshes periodically.

Connect wallet — "Connect Wallet" opens a modal with supported wallets. After a successful connection, the modal closes automatically. When connected, the header shows a truncated address and a dropdown with Account and Disconnect.

Account (when connected) — "Account" opens a sidebar with: avatar (upload, stored per wallet); username (edit and save); SOL balance; connected wallet address; Disconnect.

Autonomous Agent Protocol — Command Center: AITerminal (scrolling logs) and AutoPilot card ("Autonomous Agent Protocol", "Auto Pilot - Your Agent Partner"). When connected: toggle, P/L, risk slider, and SOON.

AI Agent Chat (when connected) — Floating button (bot icon) at bottom-right opens the chat. Topics: portfolio, risk, market research, autonomous agent, PAP. Replies use the backend POST /api/agent/chat; the server can use LLM providers such as Groq (OpenAI-compatible), Anthropic, or OpenAI — keys are never in the app bundle.

Swap — The Swap link opens the Swap page: trading chart (1H, 4H, 1D, 1W) and manual swap form. Connect wallet, choose From/To tokens (SOL, USDC, USDT), enter amount, click Get quote (Jupiter), then Swap. Success links to Solscan.

Other sections — Dashboard (hero, Total Users card, stats strip), Research & Screening, PAP Tokenomics (supply, holders, contract link), Subscription Tiers, Roadmap, in-app Docs. Footer: Total Users and social links (X: Planktonomous).

3. Backend API

Express + TypeScript server. Default base URL: http://localhost:3000. CORS allows the frontend origin.

Endpoints

Health: GET /api/health, /api/health/live, /api/health/ready.

Research: GET /api/research/feeds, /api/research/screener.

Subscription: GET /api/subscription/tiers, /api/subscription/tiers/:id.

Agent: GET /api/agent/status, GET /api/agent/config, POST /api/agent/chat (Plankton Agent; requires server LLM keys, e.g. Groq).

Stats (user count): GET /api/stats/users — returns { count } (unique wallets connected). POST /api/stats/connect — body { wallet: "<address>" } — registers a wallet (idempotent), returns { count, isNew }.

Market: GET /api/market/token-info?mint=... for token metadata; GET /api/market/token-info?mint=...&holders=1 returns { mint, holderCount } (Jupiter index) for holder counts without adding an extra serverless route.

All responses are JSON except live health (plain text). Server port and environment-specific settings are configured via environment files; not documented here for security.

4. Integration (frontend and API)

Set the API base URL in the frontend configuration (see the project repository). Use it in fetch() calls to the endpoints above. You can replace mock data in components with API data. CORS for other origins is configured in the server.

5. Roadmap

Phases 0–3 (Narrative through Pre Launch Mainnet) are LIVE. Phases 4–8 (Security & Reliability through Full Launch) are SOON. The in-app Roadmap section shows full titles and descriptions. PAP (Plankton Autonomous Protocol) is live on Solana; token details appear under Tokenomics and Docs.
